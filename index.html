steps.js
An open source vanilla Javascript library for creating guided user flows

Vanilla Javascript, No jQuery, Light, Easy to use, Guide the user on your website, form, etc.
Tested on

  • Google Chrome (Version 60.0.3112.90 (Official Build) (32 bits))
  • Google Chrome (Version 61.0.3163.100 (Official Build) (64-bit))
  • Mozilla Firefox(46.0.1)

Demo

This demo simulate a user flow where you want to guide the user from the product selection to the "Add to cart" action.


Getting Started

First, you need to download the latest version of steps.js (minified) and place it in your Javascript assets/libs folder (e.g: /assets/js/stepsjs-1.0.0.min.js).

Then include the steps.js library in your page add the following line to the tag: 

        
<script type="application/javascript" src="/assets/js/stepsjs-1.0.0.min.js"></script>

Be sure to replace the file name by the correct one if you are using a non-minified or an older version of steps.js.

Usage

1. Define the steps

The first part of the implementation is to define the user flow. In order to do that we define a variable containing an array filled with the differents steps of the flow. Each step is an object containing the following attributes :

  • elementId: ID of the element
  • title: Title for the hint
  • description: Description for the hint
  • triggerNext: Event on the element that is going to trigger the next step (supported events: click, change)

Example: 

                
var steps = [
    {
        elementId: 'step1',
        title: 'Step 1',
        description: 'You need to click on that div to go to the next step',
        triggerNext: 'click'
    },
    {
        elementId: 'step2',
        title: 'Step 2',
        description: 'You need to change this value to go to the next step',
        triggerNext: ['change', 'keydown']
    },
    {
        elementId: 'test3',
        title: 'Step 3',
        description: 'You need to click on that button to finish the flow',
        triggerNext: ['click', 'touchstart']
    }
];

2. Run steps.js

There is different way to run steps.js.

Default

The default run will automatically start the user flow using all default options of steps.js. 

                
// Run defaults
new StepsJs(steps);

Customize options

If you'd like to personalized your instance of steps.js you can pass an object containing override properties for the available options. This will also automatically start the user flow. 

                
// Run with customized options
new StepsJs(steps, options);

Control the flow

You can also decide to have more control on the user flow by storing your steps.js instance in a variable. It will allow you to use the class methods to go to the previous or next step. This will also automatically start the user flow. 

                
// Run and store in a variable to control the process
var demo = new StepsJs(steps, options);

// Skip first step
demo.processStep('next');

3. Customize Options

steps.js allows you to customize various parameters of the StepsSj instance. They are all passed as properties of the options variable.

Here is a full example of a customize set of options: 

                
var options = {
    duration: 200,
    styles: {
        frame: {
        	customClass: 'custom-frame',
            borderWidth: 3,
            borderColor: '#ff0000'
        },
        hint: {
        	customClass: 'custom-hint',
            borderWidth: 1,
            borderColor: '#00ff00',
            borderRadius: 5,
            backgroundColor: '#dbffdb'
        }
    }
};

Here is the description of all the available options:

  • duration: Animation duration for the transition between steps in ms (e.g: 1000)
  • styles
    • frame
      • customClass: Custom css class name
      • borderWidth: Frame border width in pixels (e.g: 2)
      • borderColor: Frame border color hex code (e.g: #f6e78e)
    • hint
      • customClass: Custom css class name
      • borderWidth: Hint border width in pixels (e.g: 2)
      • borderColor: Hint border color hex code (e.g: #f6e78e)
      • borderRadius: Hint border radius in pixels (e.g: 2)
      • backgroundColor: Hint background color hex code (e.g: #f6e78e)
